# limonitor — ideas

- Auto-reconnect on BLE disconnect
- Config file so common flags don't need to be passed every run
- Alert thresholds (low SoC, high temp) with optional notify-send/osascript
- Multi-battery support (repeated -n/-a flags, split TUI panels)
- Demo mode: --demo N to simulate N batteries
- WebSocket or SSE instead of meta-refresh for live updates
- Grafana dashboard template for /metrics
- Remember last connected device
- CSV/JSON export endpoint
